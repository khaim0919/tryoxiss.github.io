Permission Stacks

// Permission stack (Higher levels take priority): // // Administrator (manage_all) will ALWAYS override EVERYTHING ELSE // Roles (Higher overrides lower, except for manage_all. See more info below) // Channel Permissions // Defult hub permissions // Defult permissions

// Roles stack: // Lets say we have someone with two roles: // // Can't Ping: // - Defult // - Can't ping anyone // // Trusted: // - Defult // - Can Ping All // // and someone has both, the can't ping anyone would override it. Any permissions left as defult (passthrough to default/hub), will "shine through".
